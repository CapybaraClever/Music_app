# Музыкальный архив (Music_app)

Десктопное GUI-приложение на C++ для удобного учёта, хранения и структурирования музыкальных композиций. Интерфейс построен на базе фреймворка **Qt 6 (Qt Widgets)**, а сборка проекта управляется с помощью кроссплатформенной системы **CMake**.

---

## Запуск готовой программы

Если вы хотите протестировать приложение без необходимости открывать исходный код и компилировать проект в IDE, вы можете скачать уже готовую скомпилированную сборку под Windows.

1. Перейдите в раздел **Releases** на главной странице этого репозитория.
2. Скачайте прикреплённый архив проекта (или перейдите по указанной там прямой ссылке на облачное хранилище).
3. Распакуйте архив в любую удобную папку.
4. Запустите исполняемый файл **`MusicArchive.exe`**.

*Примечание: Все необходимые для работы динамические библиотеки (`.dll`) и плагины Qt уже находятся внутри архива рядом с исполняемым файлом, поэтому дополнительная установка Qt на компьютер не требуется.*

---

## Инструкция по сборке из исходного кода

Если вы хотите собрать проект самостоятельно, доработать его или запустить внутри среды разработки, выполните следующие шаги.

### Требования к окружению
Для успешной сборки и запуска вам понадобятся:
* **Компилятор**: с поддержкой стандарта **C++17** и выше (MSVC 2019/2022, GCC или Clang).
* **Система сборки**: **CMake** (версии 3.16 или новее).
* **Фреймворк**: **Qt 6** (протестировано на Qt 6.11.1+), в частности модули `Core`, `Gui` и `Widgets`.
* **IDE (рекомендуется)**: Visual Studio 2022 (с установленным компонентом "Разработка классических приложений на C++") или Qt Creator.

### Пошаговый процесс подключения и сборки

#### Шаг 1. Клонирование репозитория
Склонируйте проект к себе на локальный компьютер через Git-клиент или консоль:
```bash
git clone [https://github.com/CapybaraClever/Music_app.git](https://github.com/CapybaraClever/Music_app.git)
cd Music_app
Шаг 2. Открытие проекта в Visual Studio
Запустите Visual Studio.

Выберите пункт «Открыть локальную папку» (Open a local folder).

Укажите на корневую папку, куда вы только что склонировали репозиторий (папка, содержащая главный файл CMakeLists.txt).

Visual Studio автоматически распознает проект как CMake-структуру и запустит генерацию кэша. Дождитесь в окне вывода сообщения: Создание CMake завершено.

Шаг 3. Конфигурация сборки и компиляция
В верхнем меню выберите целевую архитектуру (например, x64-Debug или x64-Release).

В выпадающем списке выберите исполняемый таргет: MusicArchive.exe.

Нажмите кнопку «Сборка» (Build) в верхнем меню ➔ «Пересобрать всё» (Rebuild All) или просто нажмите горячую клавишу F5 для компиляции и автоматического запуска отладки.

📦 Развёртывание приложения (Deployment)
При компиляции проекта CMake создаёт чистый бинарный файл внутри глубоких каталогов сборки (например, out/build/x64-Debug/Music_app/). Чтобы этот файл мог запускаться автономно вне Visual Studio, его необходимо развернуть с помощью утилиты windeployqt.

Если вы хотите собрать собственный дистрибутив:

Откройте консоль командной строки Qt (Qt Command Prompt), соответствующую вашему компилятору.

Перейдите в папку с вашим скомпилированным файлом MusicArchive.exe.

Выполните команду автоматического копирования зависимостей Qt:

Bash
windeployqt MusicArchive.exe
Утилита самостоятельно подтянет все необходимые графические плагины платформ (platforms/qwindows.dll), стили интерфейса и основные библиотеки (Qt6Core.dll, Qt6Gui.dll, Qt6Widgets.dll), после чего программу можно упаковывать в .zip и отправлять пользователям.

📂 Структура основных файлов проекта
CMakeLists.txt — главный конфигурационный файл для автоматической сборки проекта, подключения модулей Qt 6 и линковки библиотек.

main.cpp — точка входа в программу, инициализация объекта приложения QApplication и запуск главного цикла обработки событий.

Music_app.cpp / .h — логика и бэкенд центрального окна музыкального архива, обработка списков треков, фильтрация и сериализация данных.

Music_app.ui — XML-описание визуального интерфейса главного окна, созданное в Qt Designer.

addsongdialog.cpp / .h / .ui — дочернее диалоговое окно, отвечающее за добавление новых музыкальных композиций и валидацию введённых данных.

song.h — базовая структура/класс сущности песни (Song), инкапсулирующая в себе поля (название, автор, год выпуска) и поддерживающая потоковую сериализацию через QDataStream.
